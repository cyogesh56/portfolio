Problem

The legacy tool used by Continental’s audio engineers lacked usability and was inefficient. They were always running into errors, and the lack of new features make them use 3rd-party tools. The inefficiencies in the current tool and the requirement of new features formed the need to develop an enhanced version of their audio tool desktop application.

Solution

We followed the usability principles and laws to create a future-proof and scalable design language, usability guidelines for both design and development. Then we created the UI and UX for the tool. We also conducted usability tests to measure the design against the KPIs.

Result

We created an MVP, which we handed over to the Continental’s developers for production. With the redesigned tool, we were able to improve the efficiency of the tool marginally and nearly eliminate all the errors. The improved aesthetic appeal also means that this tool can now be marketed to its business partners and customers, as well.

About the Company

Continental is a German automotive manufacturing company and is the 4th largest tire manufacturer in the world, but they do not just make tires. Continental also specializes in brake systems, interior electronics, automotive safety, powertrain and chassis components, tachographs and other internal automotive parts. About the Project

Continental approached DePaul Innovation Development Laboratory (DePaul iD Lab) to redesign their audio tuning tool with overhauled UI, UX, and additional features. The project was a collaboration between DePaul iD Lab cross-functional team and Continental front and back-end engineers and stakeholders.

Target Audience

This tool caters to Continental’s internal use by their engineers, undisclosed clients, and individual customers.

Process

Learning the Context of use from the Engineers

We read the published documentation on audio tuning, digital signal processing, and data processing to understand the functionality of the tool. Then we invited audio engineers to demonstrate to us how they are using the current tool in their context and work environment. We discovered that:

  1. This tool has a very complex architecture due to the functionality it provides. There will be a point where a reduction in complexity will not be possible.

  2. Users might use it under direct sunlight, which would cause screen glare and would reduce the visibility of the on-screen content.

  3. This tool can also be used during the night while driving the car and under very variable work conditions.

  4. The current tool does not ask for confirmation from the user for any action, potentially leading to catastrophic results if the user performs the wrong move.

Following the User-Centered Design Process

I focused on to follow the UX principles to create a refined design. The design team followed 7 UX principles:

  • Consistency: Create a unified and cohesive experience throughout the tool so that if the user uses moves between features of the tool, it looks and behaves the same way.

  • Usefulness: Make the tool more usable by providing the context-aware content to the user and hide all the non-important and non-relevant things from the screen so that the user can focus on the task at hand.

  • Communication: Provide easy to find and easy to understand visual cues to the user and provide appropriate, transparent, and timely feedback to the user so that the user stays informed of what is happening of his actions on the screen.

  • Simplicity: Reduce clutter and remove all the unnecessary items from the screen so that it decreases the cognitive load and increases the focus of the user on the task at hand.

  • Error Prevention and Handling: Ask for confirmation before allowing any potentially destructive actions of the user to help prevent any catastrophic consequences in critical situations.

  • Efficiency: Accommodate both new and experienced users to help them reach the same level of efficiency and expertise level by easing the tasks in the tool.

  • Work Load Reduction: Use all the above design principles to make user’s work more comfortable, simpler, faster, and efficient.

Conceptualizing using Sketches

The challenge was to create a design that can be learned by new users who have no idea what this tool is and how can they use it; and their engineers who exactly know where everything is located in the tool. My idea was to use a conversational wizard-style interface that users and engineers can talk to and get their work done. However, it was not what the client was looking for. They were looking for a design that looks and behaves exactly the same for both new users and their engineers. The reason being, to make new users efficient in using the real interface rather than talking to a robot and achieving nothing. So I went back to the drawing board and created a lot more designs.

Redefining the User Experience

Not just the UX but also the UI and taxonomies were also simplified for the user to understand and get going with the tool smoothly. Some of the changes we did:

  1. Visual enhancement: Enhanced the interface design to make it more visually appealing.

  2. Color Contrast enhancement: I tested a few variations of the primary color schemes to improve the color contrast throughout the tool to enhance the visibility of UI elements in varying lighting environments.

  3. Simplifying Iconography: I redesigned 90% of the icons to better align them with the functionality of the associated feature in the tool. We tested the icons during the usability test to find out how the engineers interpret them.

  4. Reducing space wastage: With improved visual design and confident iconography, I was able to reduce the amount of space used by each UI element on the screen, still keeping the font size and icon size precisely the same as before.

  5. Reclaiming space: More screen space was reclaimed by removing the text labels and putting them as either tooltips or expandable sidebar. The usability test indicated that all the engineers were able to quickly understand the meanings of icons correctly without any given context.

  6. UX Copyrighting: UX Copy was an essential part of the entire redesign. Text everywhere was rewritten, wherever required, to make it easier for the user to understand what is going on, what they have to do, and what will be the potential outcome of their actions.

We initially tested out all the abovementioned changes in few of the screens to test the effect of the change on engineers and after finding out that the changes are helping them improve the way they interact with the tool, we updated rest of the screens and used that design to create new screen and features for the tool.

How might we have the tool look the same in both design and development?

How might we make it easier for future developers to have the same design as us?

We initially started without a design system, but soon we realized that neither the designs are not matching exactly, nor the development looks the same as design. So, we decided to co-create a mutual design system among design and development with shared specifications.

How might we make the tool speak the language of the user?

Since the target users for this tool are primarily the engineers from Continental, the tool has to speak and work in their language, plus it has to be easy for new users to get on board and start learning. For the product to speak the user’s language, it has to be programmed to work that way, and it is a UXer’s job to provide the specifications to the developers on how it can work in a user-friendly way.

Test, Validate and Fix

  • 4 Usability Tests (Formative to Summative)

  • 5 Users (Continental’s Audio Engineers)

  • 1 hr. duration each

Testing Metrics

  • Ease of Use

  • Efficiency

  • Error Rate

  • Findability

  • Learnability

Final Product

The final product is a Windows application that is currently being developed by Continental’s front-end and back-end developers. The design guidelines we created will be used by their team to design new features for future releases. One of the biggest challenges of this project was to use the legacy technology to create modern visual elements with it, to make the product both backward and forward compatible, i.e., this tool is developed using C# Winforms with every element custom made to cater users of both Windows 7 and Windows 10 without any aesthetical changes.

Achievements

We achieved a high level of UX standards on this project by following the user-centered design process and UX principles. With the accomplishment of this project, we achieved the following things:

  • Improve the aesthetic appeal of the tool to cater to it better toward Continental’s business partners and customers.

  • Handed-over a complete and future-proof front-end and design system for their developers.

  • Made the tool easy to use, easy to learn, and faster to complete tasks based on our usability test results.

  • Improve the efficiency of the engineers by 2x, thereby also increasing the efficiency of the tool.

  • Reduce the error rate in the tool by more than 50% by following the UX principles and our creative problem-solving ability.

Learnings

  • Co-creation is the key to streamline the workflow for both designers and developers in terms of how much time they spend creating components.

  • Testing early and testing dirty is the way forward to steering the project in the right direction.
