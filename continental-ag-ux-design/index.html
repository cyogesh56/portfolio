How might we streamline and enhance the user experience of a highly technical audio tuning application, improving efficiency, and ensuring ease of learning?

Nov 2018 - Mar 2019

Engineers and business partners at Continental depend on audio tuning software to guarantee optimal clarity of microphones and speakers across various environments and vehicles. As a UX Design Intern, I spearheaded the project to redesign the tool to ensure better onboarding, efficiency and feedback-based outputs.

01. Identifying the Challenge

Scenario

Engineers developed this audio tuning tool specifically for testing microphone and speaker audio in various environments, ranging from noisy roads to adverse weather conditions. The goal is to ensure clear and audible communication for both passengers (including the driver) and listeners.

Problem

  • The tool was created several years ago without any UX intervention.

  • Primary focus for the experience was to be efficient and quick rather than user-friendly or easy to use.

  • As the feature set grew and the code got complex over time, the software started generating numerous errors unknown to the new users, requiring specialist assistance.

  • The tool was later adopted by Continental’s partners requiring improved user experience and visual overhaul for easy onboarding or new users.

02. Designing a Curated Experience

The Ask

“It needs to be easy to learn for a new user yet should not interfere with an expert’s usage of this tool"

- Team

About the tool

While the phrase "learn how to use this tool" may not sound like the ideal user experience, it is important to consider the complexity of the features offered by this tool. It is designed for users who possess knowledge of digital signal processing and who either work at Continental or with affiliated partners. The tool can:

  • Work with digital signal processing to tune the audio for a specific car based on proprietary options and parameters.

  • Create, save and load configurations.

  • Genrate reports of all the modified and unmodified changes on demand.

Conceptual sketch of an early version of wizard interface for onboarding new users.

How might we ensure both new and expert users can get work done with ease?

Ensuring ease of use for both new and expert users is challenging for a tool like this due to its complex functionality. However, there is an old concept that comes to mind:

"Wizard"

A wizard helps onboard new users while maintaining the functionality for experts. Additionally, expert users can also utilize the wizard to quickly accomplish tasks if necessary.

An illustrated screenshot of the wizard-style portion of the tool.

How might we make sure consistency from design to development?

  • Created a Design System to ensure the UI looks and behaves consistent across various sections of the tool, and from design to development.

  • Utilized Atomic Design System to create various pieces of user interface and interaction design for consistency across elements and pages.

  • Collaborated with the front-end developer to create components and specifications.

  • Provided designs in Adobe XD and Axure that could be inspected for developer requirements, making the job easier.

  • Once our work was completed, transferred ownership of all the specifications and documentation to the developers at Continental.

  • Conducted Knowledge Transfer sessions with Continental’s developers to onboard them with the new design system and development environment.

03. A Refined Experience to Last

  1. Visual Overhaul: There are no random colored elements that obscure text, and icons are now accompanied by text. Additionally, several other visual changes have been made.

  2. Color Contrast Enhancement: After careful consideration, blue color was chosen as the optimal color for a variable lighting environment. As a result, all buttons and important interface elements have been designed using the blue color.

  3. Simplifying Iconography: Most of the icons have been redesigned and micro-interactions have been added. Each icon was tested by engineers for interpretation and achieved a 100% success rate.

  4. Reducing Space wastage: The amount of space used by each UI element on the screen has been reduced while maintaining the same font and icon sizes as before.

  5. Reclaiming Space: To provide more screen space, text labels have been removed and placed as tooltips or in an expandable sidebar.

  6. UX Copyrighting: All text has been rewritten to enhance user understanding of the system, including clear explanations of what is happening, what actions need to be taken, and the potential outcomes of those actions.
